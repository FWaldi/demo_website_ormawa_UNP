Project Documentation: Portal Informasi ORMAWA Universitas
Dokumen ini memberikan gambaran komprehensif dari proyek Portal Informasi ORMAWA Universitas, mulai dari konsep awal hingga arsitektur teknis dan detail implementasi yang diusulkan.

1. Project Overview and Architecture
1.1. Initial Idea (from ide_awal.txt)
Portal Informasi ORMAWA Universitas adalah sebuah platform terpadu yang berfungsi sebagai pusat informasi dan dokumentasi seluruh organisasi mahasiswa di universitas, termasuk BEM, unit kegiatan mahasiswa (UKM), dan berbagai himpunan lainnya.

Website ini bertujuan untuk:

Menyediakan informasi terpusat tentang berita, kegiatan, dan profil ormawa.

Memudahkan mahasiswa menemukan organisasi sesuai minat.

Memberikan setiap ormawa kendali untuk mengelola halaman dan kontennya secara mandiri melalui dashboard admin terbatas.

Menyajikan informasi dalam bahasa Indonesia dengan desain modern, responsif, dan mudah diakses.

Struktur Navigasi Utama Website:

Landing Page: Hero image fullscreen (gedung universitas), navigation bar transparan di bagian atas berisi logo universitas, nama universitas, dan menu Beranda | Tentang | Berita | Ormawa | Kegiatan.

Tentang: Informasi singkat portal dan tujuan; slider berita penting ormawa.

Berita: Berita terkini ditampilkan sebagai highlight di bagian atas, diikuti kartu-kartu berita lama.

Ormawa: Kartu ormawa berisi logo dan info singkat; klik kartu membuka profil ormawa.

Kegiatan: Daftar acara mendatang dengan kalender interaktif.

Profil Ormawa (navigasi samping slide-in):

Beranda (deskripsi singkat dan video profil YouTube)

Tentang (visi, misi, divisi, prestasi)

Anggota (kartu profil pengurus dan anggota)

Kegiatan (jadwal acara organisasi)

Dokumentasi (galeri foto/video dan berita kegiatan lampau)

Kontak (form pesan + info kontak)

Admin Dashboard:

Setiap ormawa memiliki admin sendiri (multi-tenant, bukan super admin).

Login melalui URL khusus yang tidak ditampilkan di halaman publik.

Admin hanya dapat mengelola konten ormawanya.

Teknologi yang Diusulkan:

Frontend: TypeScript + Tailwind CSS

Backend: Node.js (Express) dengan MySQL

Kontainerisasi: Docker (lokal, sebelum deployment ke server kampus)

1.2. System Architecture (Proposed)
Portal ini dirancang dengan arsitektur multi-tenant web application yang memisahkan modul publik (informasi) dan modul privat (admin dashboard).

Frontend (Web Application)

Dibangun dengan TypeScript untuk keamanan tipe dan maintainability.

Menggunakan Tailwind CSS untuk desain responsif dan cepat.

Routing halaman publik dan profil ormawa dilakukan melalui client-side routing.

Authentication

Sistem login untuk admin ormawa dengan hashing password (bcrypt).

JWT-based authentication untuk sesi admin.

Backend (Node.js, Express, Sequelize ORM)

Sequelize sebagai ORM untuk MySQL.

API REST untuk:

Mengambil data publik (berita, kegiatan, daftar ormawa).

CRUD data ormawa oleh admin terkait.

Middleware RBAC untuk membatasi akses sesuai role (admin ormawa vs super admin).

Database (MySQL)

Tabel utama:

ormawa – data organisasi.

users – akun admin ormawa.

news – berita.

events – kegiatan.

members – anggota ormawa.

gallery – dokumentasi.

Relasi satu-to-banyak antara ormawa dan tabel konten.

Deployment (Development Stage)

Docker Compose untuk menjalankan:

Container Node.js backend.

Container MySQL database.

Container untuk frontend build (jika perlu SSR/SSG).

2. Backend Infrastructure
Komponen Utama:

Express Server (backend/)

server.js: Entry point aplikasi backend.

routes/: Routing API publik dan admin.

models/: Skema database dengan Sequelize.

middleware/auth.js: Middleware untuk validasi JWT.

Database MySQL

Dijalankan di container Docker.

Backup otomatis harian.

RBAC Middleware

Membatasi admin agar hanya bisa mengelola data ormawanya sendiri.

3. Frontend Structure
Direktori Utama

src/pages/: Halaman publik (Landing, Tentang, Berita, Ormawa, Kegiatan).

src/components/: Navbar, Footer, Card Ormawa, Card Berita, Calendar Component.

src/admin/: Halaman dashboard admin ormawa.

src/services/: API service untuk komunikasi ke backend.

Core Functionality

Landing Page: Hero image, nav transparan, CTA.

Berita Page: List berita + filter.

Ormawa Page: Grid kartu ormawa + filter kategori.

Kegiatan Page: Kalender interaktif.

Profil Ormawa: Navigasi samping + konten modular.

Admin Dashboard: CRUD konten ormawa.
